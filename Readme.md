# Network Protocol Simulation - IoT Temperature Monitoring System

## Overview
This project simulates an IoT temperature and humidity monitoring system for a server room. Data is generated by a sensor node and traverses a 5-layer network stack to reach a central monitoring server. It demonstrates how realistic network conditions affect data delivery and showcases KPI improvements.

## Network Stack Implementation
- **Application Layer:** Generates JSON formatted sensor readings (temperature and humidity). Payload size is typically around 1024 bytes per reading.
- **Transport Layer:** Packages data with source/destination ports (e.g., Source Port: 8080, Destination Port: 9090), sequence numbers, and checksums for error detection.
- **Network Layer:** Handles IP addressing (e.g., Source IP: `192.168.1.1`, Dest IP: `192.168.1.5`) and uses a BFS algorithm for shortest-path routing (e.g., Node 1 -> Node 4 -> Node 5).
- **Data Link Layer:** Frames packets with source/destination MAC addresses (e.g., Source MAC: `00:00:00:00:00:01`, Dest MAC: `00:00:00:00:00:04`) and CRC for integrity verification.
- **Physical Layer:** Modulates frames into signals using BPSK and simulates wireless transmission across an Additive White Gaussian Noise (AWGN) channel (e.g., SNR = 15 dB).

## Simulation Metrics (KPIs)
The simulation evaluates the network's performance by measuring:
- Throughput
- Average Latency
- Packet Loss
- Jitter
- Success Rate
